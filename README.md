# 算法面试

## 算法面试不是高考。

把这个问题看作是和面试官一起探讨问题的解决方案。

问题的细节和应用环境，可以和面试官沟通。

“正确“是相同的。

### 问题：我们需要对一组数据进行排序。

排序快速的确是快，但是不一定是满意的。可能我们在遇到问题，都不是最优的。

这组数据有什么特征？

#### 有没有可能包含有大量重复的元素？

#### 是否是真的近乎排好序了？

#### 是否数据的取值范围非常有限？

#### 是否需要稳定的排序？

#### 数据存储结构？归并排序

#### 大小是否可以装在内存中？
